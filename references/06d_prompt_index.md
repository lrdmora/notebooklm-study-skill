# Reference 06d — PROMPT_Index File Template
## The 15th prompt file — audits whatever is in the notebook and builds
## or repairs 00_MASTER_INDEX. Run this when files were uploaded without
## going through the Phase 1 naming process, or when the index is missing.

---

## PROMPT_Index.md

```
# PROMPT_Index — [CLASS NAME]

When invoked, audit all sources currently in this notebook and
generate or repair the 00_MASTER_INDEX. Run this before any other
prompt if files were uploaded without using the standard prefix
naming system, or if 00_MASTER_INDEX is missing or incomplete.

---

STEP 1 — AUDIT ALL SOURCES

List every source currently uploaded to this notebook.
For each source examine its contents and determine:

SOURCE NAME: [exact name as it appears in NotebookLM]
FILE TYPE: textbook / glossary / formula sheet / syllabus /
           sample test / study guide / notes / other
CONTENT SUMMARY: 1-2 sentences describing what is in this file
UNITS OR CHAPTERS COVERED: which units or chapters does this
  source cover? List them. If the source covers the full course,
  say FULL COURSE.
RECOMMENDED PREFIX: what prefix should this file have based on
  the naming system below?

PREFIX SYSTEM FOR REFERENCE:
DATA_U[#]_    Textbook or study material for a specific unit
DATA_C[#]_    Textbook or study material for a specific chapter
DATA_FULL_    Textbook or study material covering the full course
VOCAB_U[#]_   Glossary or key terms for a specific unit
VOCAB_C[#]_   Glossary or key terms for a specific chapter
VOCAB_FULL_   Glossary covering the full course
FORMULAS_U[#]_ Formula sheet for a specific unit
FORMULAS_C[#]_ Formula sheet for a specific chapter
FORMULAS_FULL_ Formula sheet covering the full course
REF_          Syllabus, sample test, practice exam, grading guide
PROMPT_       Prompt instruction files (do not rename these)
00_MASTER_INDEX The index file (do not rename this)

---

STEP 2 — FLAG ISSUES

After auditing all sources, flag any of the following:

UNPREFIXED FILES: Sources that do not follow the naming system.
  List the current name and the recommended rename.

AMBIGUOUS FILES: Sources where the content type is unclear.
  Describe what makes them ambiguous.

MISSING COVERAGE: Units or chapters that appear to have no
  DATA source. List them so the user knows what is missing.

MISSING VOCAB: Units or chapters with DATA but no glossary source.

MISSING FORMULAS: Units or chapters with DATA but no formula source
  (only flag if the class appears to have formula content).

DUPLICATE COVERAGE: Multiple sources covering the same unit or
  chapter. Flag them so the user can decide which to keep.

---

STEP 3 — GENERATE OR REPAIR 00_MASTER_INDEX

Using the audit above, generate a complete 00_MASTER_INDEX.
If 00_MASTER_INDEX already exists in the notebook, compare it
to the audit and note what needs to be updated.

Output the full index in this format:

================================================================
MASTER INDEX — [CLASS NAME]
Generated by PROMPT_Index audit
================================================================

HOW TO USE THIS INDEX
This file is read before every response. It maps your sources
to the prefix system so the study prompts can find them correctly.

To use a prompt: "Use PROMPT_[Name] for [Unit X / Chapter X /
the full course]"

================================================================
SOURCE AUDIT RESULTS
================================================================

CURRENT NAME          | TYPE       | COVERS    | RECOMMENDED PREFIX
----------------------|------------|-----------|-------------------
[one row per source]

================================================================
RENAME RECOMMENDATIONS
================================================================

Rename these files before your next session if possible.
If you cannot rename them, the prompts will still work —
they will use this index to find the correct source.

CURRENT NAME → RECOMMENDED NAME
[list all renames]

================================================================
SOURCE FILES MAPPED TO PREFIX SYSTEM
================================================================

PREFIX              | SOURCE NAME (current)         | CONTENTS
--------------------|-------------------------------|----------
[one row per source using current names or recommended names]

================================================================
GENERATED NOTES — Track Progress Here
================================================================

[ ] NOTES_U1_Extraction          [ ] REVIEW_Master_Concept_Map
[ ] NOTES_U1_Key_Concepts        [ ] REVIEW_Master_Glossary
[ ] NOTES_U1_Glossary            [ ] REVIEW_Master_Formulas
[ ] NOTES_U1_Formulas            [ ] REVIEW_Weak_Spot_Report
[ ] NOTES_U1_Slide_Deck_Outline  [ ] REVIEW_Final_Slide_Deck_Outline
[ ] NOTES_U1_Study_Notes         [ ] REVIEW_Cumulative_Study_Notes
[ ] NOTES_U1_Practice_Quiz       [ ] REVIEW_Cumulative_Practice_Exam
[expand for each unit found in the audit]

================================================================
FLAGS AND WARNINGS
================================================================

[list all issues found in Step 2]

---

STEP 4 — ADVISE THE USER

After generating the index, tell the user:
1. Whether the notebook is ready to use as-is or needs renames
2. Which units or chapters have complete source coverage
3. Which units or chapters are missing sources
4. What to run next — if the notebook looks ready, recommend
   starting with: "Use PROMPT_Extraction for Unit 1"
   If renames are needed first, list them clearly.

Note: Even if files are not renamed, all PROMPT_ files will
attempt to locate sources using this index as a fallback map.
Renaming is recommended but not required to use the system.
```
