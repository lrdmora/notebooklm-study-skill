Phase 1 — Master Index Setup

Run in Claude · Before NotebookLM
Run this in Claude before you touch NotebookLM. Paste the prompt below, fill in your class details, and Claude will rename all your files and generate your 00_MASTER_INDEX. Save that output as a file and upload it to NotebookLM along with your renamed source files.

Master Index Generator

Claude
Fill in every [bracket] before pasting. If Claude has access to your folder, paste the path. If not, list your files manually below the prompt.

NotebookLM System Prompt

Paste into NotebookLM instructions panel
Paste this into the NotebookLM instructions panel after uploading all your source files. Fill in [CLASS NAME] at the top. Do not paste the Master Index here — the system reads 00_MASTER_INDEX directly as a source file.

System Prompt

NotebookLM
→ Paste into: NotebookLM → Instructions panel
You are a study assistant for [CLASS NAME].

Your knowledge base uses a structured prefix system with two categories:
source files uploaded by the student and generated notes created during
the study workflow. Always read 00_MASTER_INDEX before answering any
question. It is the single source of truth for how this notebook is organized.

---

PREFIX SYSTEM

SOURCE FILES
- DATA_U[#]_ — Textbook and study material for that unit
- DATA_C[#]_ — Textbook and study material for that chapter
- VOCAB_U[#]_ / VOCAB_C[#]_ — Glossary. Every term is a potential
  test target. This is your anchor for all study outputs.
- FORMULAS_U[#]_ / FORMULAS_C[#]_ — Formula reference
- REF_ — Syllabus, exam weights, sample tests. Determines what is testable.

GENERATED NOTES
- NOTES_U[#]_Extraction — Foundation note for the unit. Read this first.
- NOTES_U[#]_Key_Concepts — Hierarchical concept outline
- NOTES_U[#]_Glossary — Prioritized glossary cross-referenced with DATA
- NOTES_U[#]_Formulas — Formulas cross-referenced with VOCAB and DATA
- NOTES_U[#]_Slide_Deck_Outline — Slide deck outline
- NOTES_U[#]_Study_Notes — Narrative study guide
- NOTES_U[#]_Practice_Quiz — Practice quiz
- REVIEW_Master_Concept_Map — Full course concept map
- REVIEW_Master_Glossary — Combined glossary with cross-unit priority flags
- REVIEW_Master_Formulas — Combined formula sheet
- REVIEW_Weak_Spot_Report — Gap analysis across all units
- REVIEW_Final_Slide_Deck_Outline — Full course slide deck
- REVIEW_Cumulative_Study_Notes — Full course study guide by theme
- REVIEW_Cumulative_Practice_Exam — Full course practice exam

---

SOURCE HIERARCHY
1. REVIEW_ notes — for multi-unit questions
2. NOTES_U[#]_ generated notes — for unit-specific questions
3. VOCAB_U[#]_ / FORMULAS_U[#]_ — for definitions and formulas
4. DATA_U[#]_ — for context and explanation
5. REF_ — for exam weight and validation

---

PRIORITY SYSTEM
- HIGH — In VOCAB + DATA + REF sample questions. Always include.
- MEDIUM — In VOCAB + DATA, not in REF. Include in most outputs.
- LOW — In VOCAB with little DATA coverage. Flag as a gap.
- CROSS-UNIT HIGH — HIGH in two or more units. Near-certain exam content.
- FOUNDATIONAL — Introduced earlier, carries forward. Flag when it reappears.

---

TEST QUESTION GENERATION
Default: 80% Multiple Choice / 15% Short Answer / 5% Essay

Weight by priority:
- HIGH and CROSS-UNIT HIGH → all three sections
- MEDIUM → multiple choice and short answer only
- LOW → multiple choice only if included
- Formula-linked → always include at least one application question

Multiple Choice: 4 options (A/B/C/D), one correct answer, plausible
distractors. Include correct answer and brief explanation.

Short Answer: 2-4 sentences. Definitions, comparisons, cause-effect.
Include model answer.

Essay: Claim, evidence, conclusion. Cumulative exams must synthesize
across at least two units. Include rubric (3/2/1/0 point scale).

---

BEHAVIOR RULES
- Read 00_MASTER_INDEX before every response
- Cite sources in every response
- Prefer NOTES_ files over raw DATA_ when they exist
- Check REF_ first for exam weight and priority topics
- When a student answers incorrectly, explain why using source material
- Lead with HIGH and CROSS-UNIT HIGH when asked what to study
- Always flag material gaps
- If a NOTES_ file is missing, prompt the student to run
  PROMPT_Extraction for that unit first
- If source files appear unprefixed or unrecognized, prompt the
  student to run PROMPT_Index, save the output as a Note, then
  bring that Note and the existing 00_MASTER_INDEX to Claude
  outside NotebookLM to merge and generate a clean updated index

PROMPT_Index — Utility Prompt

Run First · If Files Are Not Renamed
Run this if you skipped Phase 1 or uploaded raw unprefixed files. PROMPT_Index audits every source in your notebook and outputs a structured report of what it finds. Save that output as a Note, then bring it to Claude outside NotebookLM along with your existing 00_MASTER_INDEX. Claude merges the two, resolves conflicts, and generates a clean updated index. Upload the new index to NotebookLM and delete the old one. Your GENERATED NOTES progress is preserved. Invoke it by saying: "Use PROMPT_Index"

PROMPT_Index

NotebookLM
Upload PROMPT_Index.md as a source file. Then invoke it. The system does the rest.
→ Invoke with: "Use PROMPT_Index"

List every source in this notebook. For each source examine its
contents and determine:

SOURCE NAME: [exact name as it appears in NotebookLM]
FILE TYPE: textbook / glossary / formula sheet / syllabus /
           sample test / study guide / notes / other
CONTENT SUMMARY: 1-2 sentences describing the file
UNITS OR CHAPTERS COVERED: which units or chapters — or FULL COURSE
RECOMMENDED PREFIX: what prefix should this file have

PREFIX SYSTEM:
DATA_U[#]_     Textbook or study material for a specific unit
DATA_C[#]_     Textbook or study material for a specific chapter
DATA_FULL_     Textbook or study material covering the full course
VOCAB_U[#]_    Glossary or key terms for a specific unit
VOCAB_C[#]_    Glossary or key terms for a specific chapter
VOCAB_FULL_    Glossary covering the full course
FORMULAS_U[#]_ Formula sheet for a specific unit
FORMULAS_C[#]_ Formula sheet for a specific chapter
FORMULAS_FULL_ Formula sheet covering the full course
REF_           Syllabus, sample test, practice exam, grading guide
PROMPT_        Prompt instruction files — do not rename

STEP 2 — FLAG ISSUES

AMBIGUOUS FILES: files where content type is unclear
MISSING COVERAGE: units or chapters with no DATA source
MISSING VOCAB: units with DATA but no glossary source
MISSING FORMULAS: units with DATA but no formula source
DUPLICATE COVERAGE: multiple sources covering the same unit

CURRENT NAME          | TYPE       | COVERS    | RECOMMENDED PREFIX
[one row per source]

RENAME RECOMMENDATIONS
CURRENT NAME → RECOMMENDED NAME
[list all renames — renaming is recommended but not required]

SOURCE FILES MAPPED TO PREFIX SYSTEM
PREFIX              | SOURCE NAME (current)         | CONTENTS
[one row per source]

GENERATED NOTES — Track Progress Here
[ ] NOTES_U1_Extraction    [ ] REVIEW_Master_Concept_Map
[ ] NOTES_U1_Key_Concepts  [ ] REVIEW_Master_Glossary
[expand for each unit found in the audit]

FLAGS AND WARNINGS
[all issues from Step 2]

STEP 4 — ADVISE THE USER

Tell the user:
1. Whether the notebook is ready to use or needs renames
2. Which units or chapters have complete source coverage
3. Which units or chapters are missing sources
4. What to run next — if ready: "Use PROMPT_Extraction for Unit 1"

Phase 2 — Unit Setup Prompts

Run in NotebookLM · Once per unit · In order
Replace [#] with your unit number in every prompt. Run steps 1–7 in order. Do not skip ahead — each step feeds the next. Save every output as a Note with the exact title shown before moving to the next step.
1

Extraction Pass

NotebookLM
The foundation. Everything else in this unit builds from this note. If your files are not prefixed, the prompt will find them using 00_MASTER_INDEX or by searching all available sources. Confirm all 5 sections are present before moving to Step 2.
→ Invoke with: "Use PROMPT_Extraction for Unit [#]"
SOURCE RESOLUTION — Find your sources in this order before starting:

1. Check 00_MASTER_INDEX for the correct source name for this scope
2. Look for a source prefixed DATA_U[#]_ or DATA_C[#]_
3. If no prefixed source exists, search all available sources for
   content matching the unit or chapter specified — look for chapter
   headings, unit markers, or table of contents entries
4. For VOCAB: look for VOCAB_U[#]_, VOCAB_C[#]_, or any source
   containing a glossary, key terms list, or vocabulary section
5. For FORMULAS: look for FORMULAS_U[#]_, or any source containing
   formulas or equations
6. If a source cannot be found for a section, state what is missing
   and continue — do not stop

---

SECTION 1 — KEY CONCEPTS OUTLINE
Read the DATA source for the specified scope. Produce a hierarchical
numbered outline of every major concept. Mirror the textbook's header
structure. If the source is an unprefixed textbook, use the chapter or
unit number to locate the correct section.

SECTION 2 — GLOSSARY CROSS-REFERENCE
Find the VOCAB source and locate each term in DATA. If no dedicated
VOCAB source exists, extract key terms directly from DATA — look for
bolded, italicized, and defined terms and terms in callout boxes.

For each term: TERM | DEFINITION | CONTEXT (1-3 sentences from DATA) |
FORMULA LINK | PRIORITY (HIGH/MEDIUM/LOW)
HIGH = defined + explained in DATA + in REF questions
MEDIUM = defined + explained in DATA, not in REF
LOW = found in source with little explanation

List HIGH first, MEDIUM second, LOW last. End with GAPS section.
Note if terms were extracted from DATA directly vs a VOCAB source.

SECTION 3 — FORMULA CROSS-REFERENCE
Find the FORMULAS source. If none exists, scan DATA for equations
and formulas. For each formula: FORMULA | WHAT IT SOLVES | VARIABLES |
VOCAB TERMS CONNECTED | DATA EXAMPLE | REF FLAG (YES/NO)
If no formulas found anywhere, note it and skip.

SECTION 4 — TEXTBOOK SIGNALS SCAN
Extract from DATA:
- Italicized or bold terms not captured in Section 2
- Callout boxes, sidebars, highlighted or shaded blocks
- Worked examples (concept + brief description)
- End of chapter or unit review questions
- Author flagged reminders — "note", "remember", warnings

SECTION 5 — SUMMARY AND PRIORITY LIST
- CORE THEMES: 2-3 sentences on what this scope is about
- TOP 10 STUDY PRIORITIES: ranked with reason for each
- FOUNDATIONAL TERMS FROM PREVIOUS UNITS: how they connect here
- MATERIAL GAPS: terms or concepts with thin source coverage
- EXAM INTELLIGENCE: topics flagged by REF sources if available
- SOURCE NOTE: which sources were used per section; flag any missing
💾 Save as Note: NOTES_U[#]_Extraction

Phase 3 — Cross-Unit Review Prompts

Run after all units complete · In order
Do not start Phase 3 until every unit has all 7 notes saved. These prompts synthesize across all units — they only work when the full set of NOTES_U[#]_ files exists.
1

Master Concept Map

NotebookLM
→ Paste into: NotebookLM chat
Using all NOTES_U[#]_Key_Concepts notes across every unit, build a
master concept map for the full course. Identify:
1. The 5-10 core themes that run through multiple units
2. How each unit connects to the ones before and after it
3. Concepts introduced early that become more complex later —
   trace that progression explicitly
4. Any unit that stands largely alone — flag for independent study
Format as a structured outline with units as branches and shared
themes as the trunk.
💾 Save as Note: REVIEW_Master_Concept_Map
2

Master Glossary

NotebookLM
→ Paste into: NotebookLM chat
Using all NOTES_U[#]_Glossary notes across every unit, compile a
master glossary for the full course. For each term:
- Keep definition and priority rating from the unit note
- Add a UNITS field showing every unit the term appears in
- If a term appears in multiple units, write one sentence on how
  its meaning or application evolves across units
- Flag any term that is HIGH priority in two or more units
Sort alphabetically. Add a CROSS-UNIT HIGH PRIORITY section at the
end listing only the multi-unit HIGH terms.
💾 Save as Note: REVIEW_Master_Glossary
3

Master Formula Sheet

NotebookLM
Skip if the class has no formulas.
→ Paste into: NotebookLM chat
Using all NOTES_U[#]_Formulas notes, compile a master formula sheet
for the full course. For each formula: expression, plain language
explanation, which unit it comes from, any units where it reappears,
connected vocab terms from REVIEW_Master_Glossary, and whether it
appeared in any REF sample questions. End with a ranked list of
formulas most likely to appear on a cumulative exam.
💾 Save as Note: REVIEW_Master_Formulas
4

Weak Spot Report

NotebookLM
The most important output most people never think to run. Do not skip this.
→ Paste into: NotebookLM chat
Using all NOTES_U[#]_Extraction notes, compile a full weak spot
report for the course. Include:
1. Every term flagged LOW priority or GAP in any unit — listed by unit
2. Any concept in REF sample questions with thin DATA coverage
3. Any unit with significantly more gaps than others — flag the
   entire unit as needing supplemental resources
4. The top 10 weak spots to address before the final exam
Be direct. Flag everything underserved in the materials.
💾 Save as Note: REVIEW_Weak_Spot_Report

NotebookLM

Using all NOTES_U[#]_Study_Notes and REVIEW_Master_Concept_Map,

- Organize by core theme from the concept map, not by unit
- Cover every HIGH priority concept from every unit

7

Cumulative Practice Exam

NotebookLM
→ Paste into: NotebookLM chat
Using REVIEW_Cumulative_Study_Notes, REVIEW_Master_Glossary,
REVIEW_Master_Formulas, and all REF sources, generate a full
cumulative practice exam. Requirements:
- 80% multiple choice, 15% short answer, 5% essay
- Every unit must be represented — weight toward units with more
  REF coverage and higher cross-unit concept overlap
- All CROSS-UNIT HIGH PRIORITY terms must appear
- At least one formula application question per unit with formulas
- Essay questions must synthesize across at least two units
- Full answer key with explanations, model answers, essay rubrics,
  and source citations for every answer
💾 Save as Note: REVIEW_Cumulative_Practice_Exam
